# Security Policy

## Reporting a Vulnerability

The security of this project is important.

If you discover a security vulnerability or a potential security issue, please report it by **opening a GitHub issue** in this repository.

When reporting a security issue, please try to include:
- A clear description of the vulnerability
- Steps to reproduce the issue (if possible)
- The potential impact
- Any relevant logs, screenshots, or code snippets

---

## Responsible Disclosure

Please avoid publicly sharing exploit details before the issue has been reviewed and addressed.

We aim to review reported security issues as quickly as possible and take appropriate action when needed.

---

## Scope

This security policy applies to:
- The server
- The API
- Actions and reactions implementations
- Any related infrastructure or configuration provided in this repository

---

## No Guarantee

This project is provided "as is" without any guarantee of security or stability.  
Users and contributors are responsible for using it at their own risk.

---

Thank you for helping to keep this project secure.
